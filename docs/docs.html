Take notes without pen and paper - use only bits

Basic idea

BitBoard is (for now) just another web app for taking notes, but in the future it might turn into something more.

For now, it allows organizing notes into nested categories, and formatting them using Markdown, but soon it could support multiple note types, inter-note links, suggested meta-information, smart search, visualization and a lot more.

Source code

The source code of the whole app is freely available on Github.

API
/api

Below are listed all available API routes together with detailed explanations of their purpose, parameters, response types and statuses, as well as potential errors.

Authenticate user
POST /auth

Authenticates user by his email/password combination and returns a JWT API access token (long-term, lasting 30 days), which should be sent in the Authorization HTTP header with each subsequent request.

Parameters

email
string
required

User's email address, used for password recovery, important communications and sending notifications about interesting updates.

Must be unique.

Example: johndoe@mail.com
password
string
required

User's password, used together with username for API access authentication.

Minimum length: 6 characters.

Response

200

JSON response containing API access_token and user_id

application/json
{
    "data": {
        "access_token": string,
        "user_id": integer
    }
}

Errors

AuthenticationError
403
Attempted action requires authorization, which has failed.
BadParameterError
400
Request parameter is invalid.
DoesNotExistError
404
Requested object does not exist in the database.
MissingParameterError
400
Required parameter is missing from the request.

Get user profile
GET /users/self JWT

Returns active user's profile informations, some of which are considered private (eg. email) and should not be shared other users.

Parameters

No parameters.

Response

200

User profile data

application/json
{
    "data": {
        "email": string,
        "fullname": string,
        "id": integer
    }
}

Errors

No errors expected.
Edit user profile
PUT /users/self JWT

Modifies user's account data and returns the modified profile informations.

If any of the user's account properties is unchanged, it can be left out from the request.

Parameters

string
optional

password

optional

Minimum length: 6 characters.

Response

Creates a new category for the user, can be either top-level or nested. Has to have a unique name.
